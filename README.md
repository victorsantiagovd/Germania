\# Germania 🎓📉



Germania es un sistema web de gestión académica y analítica predictiva diseñado específicamente para mitigar la deserción escolar en las instituciones educativas rurales. El software optimiza la administración escolar centralizando el control de matrículas, la carga académica, los horarios y el registro de calificaciones.



El valor diferenciador de Germania radica en su motor analítico integrado: a través de un \*\*Índice de Vulnerabilidad Estudiantil (IVE)\*\* que procesa variables de asistencia diaria, rendimiento pedagógico y factores socioeconómicos en tiempo real, el sistema genera un semáforo de alertas tempranas. Esto permite a los directivos y docentes identificar a los estudiantes en riesgo de abandono escolar antes de que suceda, transformando los datos cotidianos del aula en decisiones estratégicas para garantizar la permanencia estudiantil.



\---



\## 🎯 Pilares del Proyecto



\*   \*\*Eficiencia Operativa:\*\* Digitaliza procesos manuales (como el llamado a lista y la entrega de notas), reduciendo la carga administrativa del cuerpo docente.

\*   \*\*Enfoque Predictivo:\*\* No solo registra eventos pasados; calcula activamente la probabilidad de deserción mediante una combinación lineal ponderada (el algoritmo del módulo IVE).

\*   \*\*Pertinencia Social:\*\* Resuelve una problemática crítica en el sector educativo rural colombiano, alineándose con las necesidades de gestión de datos de las Secretarías de Educación y el SIMAT.



\---



\## 🛠️ Stack Tecnológico



El proyecto se construye sobre una arquitectura desacoplada, limpia y altamente escalable utilizando las siguientes tecnologías:



\*   \*\*Backend:\*\* Python con el microframework \*\*Flask\*\* (Patrón Modular con Blueprints).

\*   \*\*Bases de Datos:\*\* \*\*MySQL\*\* para entornos locales/servidor y \*\*SQLite\*\* para fases ágiles de pruebas e integración de datos.

\*   \*\*Frontend:\*\* HTML5, CSS3, JavaScript Nativo (ES6+) para el consumo de la API mediante asincronía (`fetch`), y \*\*Bootstrap 5\*\* como framework de estilos responsivos.

\*   \*\*Pruebas (Testing):\*\* Framework `unittest` integrado de Python para el aseguramiento de la lógica del algoritmo IVE.

\## 🚀 Flujo de Trabajo en Git



Para colaborar de manera ordenada, el equipo utiliza \*\*Git Flow\*\* bajo la convención de \*\*Conventional Commits\*\*.



\### Reglas del Repositorio:

1\. Queda prohibido realizar commits directos sobre las ramas `main` y `develop`.

2\. Todo desarrollo se realiza en una rama de característica (`feature/nombre-tarea`).

3\. Para integrar código a la línea base se debe abrir un \*\*Pull Request (PR)\*\*, el cual requiere la revisión obligatoria de otro desarrollador y la aprobación técnica del equipo.
